# Security Policy

## Supported Versions

`kmux` is currently pre-1.0. Security fixes are expected to land on the latest development line rather than across multiple maintained release branches.

## Reporting A Vulnerability

Please do not open a public issue for security-sensitive reports.

Use one of these paths instead:

- GitHub private vulnerability reporting for this repository, if enabled
- A private maintainer contact channel, if one is available

Include the following when possible:

- affected commit, branch, or version
- platform and environment details
- reproduction steps
- expected impact
- any known workaround

## Disclosure Expectations

- Give maintainers reasonable time to investigate and respond before public disclosure.
- Avoid publishing exploit details, proof-of-concept code, or sensitive logs in public issues.
- If the report turns out to be a general bug rather than a security issue, it may be redirected into the normal issue workflow.
